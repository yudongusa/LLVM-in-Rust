# Unwind Compatibility Matrix

This document describes current unwind metadata guarantees in `llvm-codegen` output objects.

## Scope

- Object-level metadata emission only (not full language runtime EH interop guarantees).
- Applies to metadata generated by `src/llvm-codegen/src/emit.rs`.

## Current Guarantees

### ELF targets

- Emits `.eh_frame` section.
- Includes one baseline CIE + one baseline FDE per emitted function object.
- FDE range tracks emitted `.text` size in the object model.

Current caveats:
- CFI is baseline/minimal and not yet fully prologue/epilogue-shape-aware for all frame patterns.
- Personality/LSDAs are not fully modeled yet.

### COFF targets

- Emits `.xdata` section with baseline `UNWIND_INFO` header.
- Emits `.pdata` section with one baseline `RUNTIME_FUNCTION` record.
- Runtime-function end range tracks emitted `.text` size in the object model.

Current caveats:
- Unwind-code richness is minimal; complex frame layouts and full SEH parity are follow-up work.
- Section-local addressing model in relocatable objects is intentionally conservative.

### Mach-O targets

- No dedicated unwind metadata emission in current baseline.
- Follow-up work required for parity with ELF/COFF unwind guarantees.

## Test Coverage (Current)

`llvm-codegen` tests assert:
- Section presence for `.eh_frame`, `.xdata`, `.pdata`.
- Baseline record shape and key field sanity (CIE/FDE and UNWIND_INFO/RUNTIME_FUNCTION layouts).

## Recommended Next Steps

1. Frame-shape-aware unwind op generation from lowering/prologue facts.
2. Tooling-backed verification matrix (`readelf`, `llvm-dwarfdump`, `llvm-readobj`, `dumpbin`) where available.
3. Runtime integration tests covering unwind-through-compiled-frame scenarios.
